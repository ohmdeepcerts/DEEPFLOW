            🔐 Security Status
            ⚠️ GitHub Pages = Public URL — Anyone with the URL can open the app. Security relies on your PIN system being enabled.
            ✅ Data is safe — All business data (jobs, invoices, certs) lives in Supabase, not the HTML file. Someone with the URL sees only a login screen (if PIN lock is on).
            ℹ️ Supabase API key — The anon key in the file gives read/write access. Supabase RLS (Row Level Security) policies control who can do what. Your current setup uses allow_all policies — fine for an internal team tool, but add proper auth rules for stricter control.
            PIN Lock must be ON to prevent unauthorised access from the public URL. Go to Settings → Users to verify.

            🗂 Storage & Photos Guide
            How to View & Download Photos
            1. Go to Storage → deepflow bucket
            2. Navigate into jobs/ folder → then the job ID folder
            3. Click any file to preview or download it
            4. To download all: use the SQL snippet below to get URLs, then open each link
            How to Delete Photos/Files
            1. Go to Storage → deepflow
            2. Navigate to the file, tick the checkbox, click Delete
            3. Also run this SQL to remove the DB record (important!):
            DELETE FROM attachments WHERE storage_path = 'jobs/JOB-ID/FILENAME.jpg';
            ⚡ Quick Setup — Create Missing Tables
            Creates engineer_requests and engineer_alerts tables in one click. Safe to run even if they already exist.
            📝 Engineer Requests Setup
            Run this SQL once to create the engineer requests table (for overtime/leave requests from the portal):
            CREATE TABLE IF NOT EXISTS engineer_requests ( id text primary key, engineer_name text not null, type text not null, date text, hours numeric, rate text, job text, leave_type text, leave_from text, leave_to text, notes text, status text default 'pending', office_reply text, created bigint default extract(epoch from now())*1000 ); ALTER TABLE engineer_requests ENABLE ROW LEVEL SECURITY; CREATE POLICY "allow_all" ON engineer_requests FOR ALL USING (true) WITH CHECK (true);
            📍 Add Location Columns (run once)
            Required for engineer GPS tracking — run once if not already done:
            ALTER TABLE users ADD COLUMN IF NOT EXISTS last_lat double precision; ALTER TABLE users ADD COLUMN IF NOT EXISTS last_lng double precision; ALTER TABLE users ADD COLUMN IF NOT EXISTS last_seen bigint; ALTER TABLE users ADD COLUMN IF NOT EXISTS last_accuracy integer;

            💾 Backup & Restore Guide
            1. Go to Settings → Data and click "Export Full JSON Backup"
            2. Save the file somewhere safe (Google Drive recommended)
            3. To restore: click "Import Backup" and select the JSON file
            4. For a full Supabase backup: Database → Backups (Pro plan) or use pg_dump
